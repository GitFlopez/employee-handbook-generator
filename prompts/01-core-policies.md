# Prompt 1 — Core Policies (FREE)

## Use This For
Generating the four foundational policies every employee handbook needs: code of conduct, attendance, PTO, and remote/hybrid work.

---

## The Prompt

```
You are an HR policy writer creating an employee handbook for a real company. Write clear, enforceable policies — not generic templates.

**Company:** [Company Name]
**Industry:** [e.g., marketing agency, retail, tech startup, restaurant]
**State/Location:** [State — affects at-will language and specific laws]
**Employee count:** [Number of employees]
**Work model:** [In-person / Hybrid (X days in office) / Fully remote]
**Tone:** [Formal corporate / Friendly startup / Balanced professional]
**PTO model:** [Accrual (X days/year) / Unlimited / Bank (X days given upfront)]
**Standard hours:** [e.g., 9–5 Mon–Fri, flexible, shift-based]

Generate four policies:

---

### POLICY 1: CODE OF CONDUCT

Include:
- Professional standards and workplace behavior expectations
- Communication standards (internal and client-facing)
- Conflict of interest definition and disclosure requirement
- Social media conduct (personal accounts that reference the company)
- Confidentiality and data handling basics
- Consequences for violations (link to disciplinary policy)

Format: Policy title → Purpose statement → 4–6 numbered sections → Manager note

---

### POLICY 2: ATTENDANCE & HOURS

Include:
- Standard work hours and schedule
- Tardiness definition and procedure (what counts, how to report)
- Absence reporting procedure (who to call, how far in advance)
- Unexcused absence definition and threshold for disciplinary action
- Remote/hybrid-specific attendance expectations (if applicable)
- On-call or after-hours expectations (if applicable)

Format: Policy title → Purpose → Definitions → Procedures → Manager note

---

### POLICY 3: PTO & VACATION

Include:
- PTO model ([accrual/unlimited/bank]) and rate or terms
- How to request time off (process, lead time, approval)
- PTO blackout periods or busy-season limits (leave blank if none)
- Rollover policy (carryover cap or use-it-or-lose-it)
- PTO payout on termination (state-specific if [State] requires it)
- Sick leave vs. vacation (combined PTO bank or separate)
- Holidays — federal holidays plus any company-specific

Format: Policy title → Purpose → PTO Details table → Request Procedure → Manager note

---

### POLICY 4: REMOTE WORK POLICY

Include:
- Eligibility criteria (who qualifies for remote/hybrid)
- Home office requirements (internet speed, dedicated workspace, equipment)
- Availability and response time expectations during work hours
- Security requirements (VPN, screen lock, no public WiFi for sensitive data)
- Expense reimbursement for home office (if applicable)
- In-office days for hybrid employees and how they're scheduled
- Performance review consideration for remote employees

Format: Policy title → Purpose → Eligibility → Requirements → Manager note

---

After all four policies, add:
**EMPLOYEE ACKNOWLEDGMENT SECTION** — one paragraph employees sign confirming they've read and understood the handbook, plus a signature/date line.
```

---

## Tips for Best Results

- Be specific about your state — PTO payout on termination varies widely (California requires it; most states don't)
- If you don't have a remote work policy, set `Work model: In-person` and that section will still generate a "no remote work" policy
- Run this prompt first before the others — it sets the tone and structure the rest of the handbook follows
- For a more casual culture, use `Tone: Friendly startup` — the policies will read like Basecamp or Notion's handbook, not a law firm's
